Maptiler 문제 - WGS84 + 전체 영역(90 -90 180 -...

Feb 19th, 2010

maptiler 문제 - WGS84 + 전체 영역(90 -90180 -180)로 할 경우, 이미지 상하단이 늘어진다. - WGS84 + (00 9065)로 할 경우 웬지 뭔가가 안된다. - raster tile 로 할 경우, 이미지 크기에 의존해서 2의 배수로 타일이 안생긴다. 이미지 크기를 256 단위로 맞춰야 한다. 또한 구글 타일 시스템과는 Y 값이 반대로 들어온다. 구글 타일 시스템 zoom level = z 일 경우, (0,0) ~ (2z-1,2z-1) 타일이 생긴다. 단 스크롤을 계속 할 경우 타일 좌표는 계속 증가하거나 감소한다. getTileUrl 에서 적당히 wrap 시켜야 한다. ex> 줌레벨 3 일 경우, 타일은 (0,0) ~ (7,7) 이 생긴다. 최종 성공 버전 0. 소스 이미지를 256 배수 정사각형으로 만든다. (직사각형도 되지만 별도의 타일 크기를 하드코딩해야 한다) 1. image based tile (raster) 2. georeference: 66.513260443111850 900 3. preview 시 적도 윗쪽 아프리카+유럽이 보이는 왜곡이 적은 부분이 잡혀야 함 4. 렌더링 후 각 타일들이 2의 승수만큼 존재하는지 체크. 5. 불필요한 kml 파일들은 삭제할 것.
