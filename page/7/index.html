Do Not Remove this.element in ui.widget.destroy()

Oct 6th, 2010

ui.widget.element.remove() will call own destroy() automatically.

$.Widget.prototype = {

_createWidget: function( options, element ) {
    // $.widget.bridge stores the plugin instance, but we do it anyway
    // so that it's stored even before the _create function runs
    $.data( element, this.widgetName, this );
    this.element = $( element );
    this.options = $.extend( true, {},
        this.options,
        $.metadata && $.metadata.get( element )[ this.widgetName ],
        options );

    var self = this;
    this.element.bind( "remove." + this.widgetName, function() {
        self.destroy();
    });

    this._create();
    this._init();
},

Slick Interface Tips

Oct 2nd, 2010

검은 반투명 사각형

  • inner shadow (normal blend, white, 1px): 레이어 내부에 위에서 내려오는 라이트 살짝 넣기
  • outer glow (normal blend, black): 레이어 외부 전체에 살짝 퍼진 그림자
  • stroke (black, 1px): 레이어 외곽선 1픽셀 검은선

특정 레이어 윗쪽에 그라데이션 먹이기

  • cmd + click 로 특정 레이어 전체 선택, alt + click 으로 아래 부분을 제거
  • G 로 검은색 칠하기
  • gradient overlay 로 윗쪽을 하얗게 만들기

검은 바탕에 입체감 있는 폰트

  • drop shadow (multiply, 1px, black): 아래 방향 그림자
  • inner shadow (overlay, 1px, white): 윗쪽 방향 라이트
  • gradient overlay (gray-white): 아래쪽으로 회색 그림자
