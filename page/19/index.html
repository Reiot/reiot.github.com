map( lambda x : xx, [1,2,3,4,5] ) # 리스트를 순회하면서 제곱을 한 걸 리턴한다. map 함수는 리스트 comprehension 으로 대체할 수 있다. 위의 예제는 아래처럼 고칠 수 있다.
[ xx for x in [1,2,3,4,5] ] filter 함수는 for .. in if 으로 대체할 수 있다.
filter( lambda x: x<4, [1,2,3,4,5] ) == [ x for x in [1,2,3,4,5] if x < 4 ] generator 는 [] 대신 () 로 감싸서 for 에 넘길 수 있다.
for x in ( yy for y in [1,2,3,4,5] if yy < 10 ):

  for y in (0,1,2,3):

==> print [(x,y,x*y) for x in (0,1,2,3) for y in (0,1,2,3) if x < y] # 다만 리스트다. reduce 를 이용하면 리스트를 순회하면서 각 값들을 특정 값에 누적 연산시킬 수 있다.
result = 1

for i in [1,2,3,4,5]:

==> reduce( lambda a,b: a*b, [1,2,3,4,5] ) dictionary 생성시 { ‘a’: 1, ‘b’: 2 } 대신 dict( a=1, b=2 ) 로 만들 수 있다. C 의 a ? b : c 는 다음과 같이 할 수 있다.

b if a else c C 의 switch 문은 딕셔너리.get( key, default ) 로 대체할 수 있다.

jquery.data() - 특정 노드에 (Key, Json ) 를 저장…

Mar 5th, 2010

jquery.data() - 특정 노드에 (key, json ) 를 저장할 수 있다. - 여러 값을 저장하고 읽어오는 캐시의 역할을 한다. - 다만 HTML 안에 값이 보이진 않는다. jquery metadata plugin - 특정 노드의 속성 혹은 script type=application/json 태그 안에 json 을 읽어온다. - 기본적으로 class 속성 안에 저장한다. 물론 다른 속성에 넣을 수도 있다. - 하나의 노드에는 하나의 메타데이터만 들어간다. - HTML 코드 레벨에서 값을 볼 수 있다. 즉 서버에서 미리 준비해둘 수 있다. - 일단 파싱하면 메모리에 저장되므로 초기값을 변경해서 저장할 수 있다. (타이머 등등에서) jQuery.metadata.setType( type, name ) - 기본적인 저장 위치를 설정. 커스텀 속성에 저장할 경우 이걸 해두면 그냥 metadata() 만 호출하면 된다. .metadata() - 딕셔너리를 리턴한다. 이 딕셔너리에 값을 변경하면 그 다음번 접근에서도 변경된 값이 리턴된다.
