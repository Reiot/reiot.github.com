URL Anchor & Query Format From Http://ww…

Apr 29th, 2010

URL anchor & query format from http://www.ietf.org/rfc/rfc2396.txt URI-reference = [ absoluteURI | relativeURI ] [ # fragment ] relativeURI = ( net_path | abs_path | rel_path ) [ ? query ] 즉 #anchor 는?query 다음에 오는게 맞다. http://somewhere.com/some/where/?query+string#anchor 아무래도 쿼리 문자열로 인해서 리소스의 유일성이 달라질 수 있으므로, 앵커의 우선순위는 쿼리보다 낮다고 본다는 뜻이리라.
